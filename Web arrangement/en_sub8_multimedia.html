HTML5 makes the web richer. This is because you can now put in audio and video as well as text and images. You can also put background music on your personal home page or upload videos of yourself to share them. Not only that, but it's also possible
to put subtitles in the video with HTML5.

HTML5 Multimedia

object embed audio vidio source track WebVTT

<object>

<object>태그는 외부 파일을 삽입하는 태그
 - 기본형은 <object data="경로" type="유형" [name="이름" width="너비" height="높이"]></object>
- <object>태그에 사용되는 속성들은 다음과 같음
속성 설명
data 외부 파일의 경로를 지정합니다.
type 포함시킨 내용의 유형을 지정합니다.
name 다른 요소들과 구분할 수 있는 이름을 지정합니다.
width 포함시킨 내용의 너비 값을 지정합니다.
height 포함시킨 내용의 높이 값을 지정합니다.

<embed>

<embed>태그는 외부 파일을 삽입하는 태그
 - <embed>태그도 웹 브라우저에서 재생할 수 없는 콘텐츠를 웹 문서에 삽입할 때 사용
- 기본형은 <embed src="경로" type="유형" width="너비", height="높이">
- 단일 태그

<audio>

<audio>태그는 오디오 파일을 삽입하는 태그
 - 배경 음악이나 효과음 등을 삽입할때 사용
- 대부분 mp3를 사용
- 기본형은 <audio src="오디오 파일 경로" [속성] [속성="속성 값"]></audio>
- <audio>태그에 사용되는 속성들은 다음과 같음
속성 설명
autoplay 오디오를 자동 재생합니다.
controls 웹 화면에 컨트롤 막대를 표시합니다. 컨트롤 막대에는 재생/멈춤, 진행 바, 볼륨 등이 표시됩니다.
loop 오디오를 반복 재생합니다.
muted 오디오를 재생해 진행하지만 소리는 끕니다.
preload 재생 버튼을 눌러 재생하기 전에 오디오 파일을 다운로드해 준비해 둡니다.
- <audio>태그와 <vidio>태그에 사용되는 속성들은 거의 일치함(<vidio>태그에도 사용 가능)
속성 설명
width, height 비디오의 크기를 조절합니다. width나 height 값 하나만 지정하면 나머지는 자동으로 게산되어 표시됩니다.
controls 미디어 파일에 컨트롤 막대를 함께 표시합니다. 속성 값은 따로 없으며 contorls이라는 속성만 쓰면 됩니다.
preload 파일을 재생하기 전에 미디어 파일을 모두 다운로드할 것인지, 일부 정보만 다운로드할 것인지 여부를 지정합니다.
none 미디어 파일을 미리 다운로드하지 않고 사용자가 재생 버튼을 눌러야만 다운로드하기 시작합니다.
metadata 미디어 파일을 즉시 사용하지 않고 사용자가 재생 버튼을 눌러야만 다운로드하기 시작합니다.
auto 사용자가 득시 이용할 수 있도록 웹 문서를 로드할 때 미디어 파일도 모두 다운로드합니다. 다만, 다운로드가 끝나도 사용자가 재생 버튼을 눌러야만 재생됩니다. 속성 값을 지정하지 않고 preload라고만 설정하면 auto 값을 기본으로 합니다.
muted 비디오를 재생할 때 소리는 끄고 화면만 재생됩니다. 비디오를 문서 배경으로 사용하거나 특별히 소리가 필요하지 않은 비디오일 때 사용합니다.
autoplay 특정 값 없이 추가하면 비디오나 오디오 파일을 다우로드하자마자 재생됩니다. 하지만 이 속성을 사용하더라도 모바일 기기에서는 자동 재생되지 않습니다. 또한 크롬 브라우저의 경우에도 무분별한 광고 등 을 막기 위해 멀티미디어가 자동 재생되지 않습니다. 가능하면 HTML5에서 자동 재생은 사용하지 않길 권합니다.
loop 미디어 파일 재생이 끝났을 때 파일 맨 앞으로 돌아가 반복 재생합니다. 특정 값이 없습니다.
poster 브라우저 문제나 인터넷 연결 문제 등 비디오를 재생할 수 없을 경우, 비디오 화면 자리에 대신 표시하는 이미지를 '포스터 이미지'라고 합니다. poster속성으로 화면에 표시할 이미지를 지정할 수 있습니다.

<vidio>

<vidio>태그는 비디오 파일을 삽입하는 태그
 - 기본형은 <vidio src="비디오 파일 경로" [속성] [속성="속성 값"]></vidio>
- <vidio>태그와 <audio>태그에 사용되는 속성들은 거의 일치함(<audio>태그에도 사용 가능)
- <vidio>태그에 사용되는 속성들은 다음과 같음

<source>

<source>태그는 여러 미디어 파일을 한꺼번에 지정하는 태그
 - <source>태그에 사용되는 속성들은 다음과 같음
속성 설명
src 미디어 파일의 경로를 지정하는 필수 속성으로 파일 경로를 자정할 때는 경로에 공백이 있으면 안됩니다.
type 웹 브라우저가 해당 미디어 파일을 재생할 수 있는지 여부를 확인할 수 있도록 미디어 파일의 유형을 알려줍니다.
codecs 비디오 코덱을 지정합니다.

<track>

<track>태그는 비디오 화면에 자막을 추가하는 태그
 - <track>태그에 사용되는 속성들은 다음과 같음
속성 설명
kind 자막의 종류를 지정합니다. 사용할 수 있는 값은 다음과 같으며 생략할 경우, subtitle로 간주합니다.
subtitle 자막입니다. 소리를 킬 수 있지만 이해할 수 없는 경우, 다른 언어로 번역한 자막일 경우에 사용하며 비디오 화면에 표시됩니다.
caption 캡션입니다. 청각장애인용 자막이거나 소리를 들을 수 없거나 켤 수 없는 경우에 사용합니다. 비디오 화면에 표시됩니다.
descriptions 비디오 콘텐츠에 대한 설명입니다. 비디오 화면에는 표시되지 않습니다.
chapters 비디오 탐색을 위한 장 제목입니다. 비디오 화면에는 표시되지 않습니다.
metadata 비디오 콘텐츠 정보입니다. 비디오 화면에는 표시되지 않습니다.
src 자막 텍스트의 파일 경로를 지정합니다.
srclang 사용한 언어를 지정합니다. kind속성 값이 subtitle이라면 반드시 지정해야 하는데 en이나 ko처럼 언어를 나타내는 약자로 표기합니다.
label 자막이 여러 개일 경우, 자막을 식벽할 수 있도록 제목을 달아 줍니다.
default 자막 파일이 여려 개일 경우, 기본으로 사용할 자막을 default로 지정할 수 있습니다.
