웹 폼은 이름에서도 알 수 있듯이 특정 항목 에 사용자가 뭔가를 입력하는 형태로 우리가 익숙하게 사용하는 로그인 창이 바로
웹 품을 이 용한 대표적인 사례입니다. 웹 폼을 이용하면 로그인 창에 아이디와 비밀번호를 입력한 것처 럼 이메일 주소, 전화번호
등의 항목을 넣을 수 있고 체크박스를 넣을 수도 있습니다. 이렇게 다양한 항목들은 HTML5가 지정되고나서 사용할 수 있게
되었는데요. 아직까지 브라우저별 로 지원 내용이 다르기 때문에 표준 규약이 완성될 때까지는 하나씩 확인해야 합니다.

<datalist>

<datalist>태그는 목록 중 데이터를 선택할 수 있는 태그
 - <datalist>태그는 드롭다운에서 데이터 목록 중에서 값을 선택하도록 만듦
- 이때 <option> 태그의 속성은 다음과 같음
속성 설명
value 사용자가 레이블을 선택했을 때 서버로 넘겨질 값을 지정합니다.
label 사용자를 위해 브라우저에 표시할 레이블 입니다. 따로 지정하지 않을 겨우, value 값을 레이블로 사용합니다.

<textarea>

<textarea>태그는 여러 줄 입력하는 텍스트 영역을 만드는 태그
 - <textarea>태그는 한 줄 이상의 문장을 입력할 때 사용
속성 설명
name 다른 폼 요소와 구분하기 위해 텍스트 영역의 이름을 지정합니다.
cols 텍스트 영역의 가로 너비를 문자 단위로 지정합니다.
rows 텍스트 영역의 세로 길이를 줄 단위로 지정합니다. 지정한 숫자보다 줄 갯수가 많아지면 스크롤 막대가 생깁니다.

<button>

<button>태그는 버튼을 만드는 태그
 - button의 type 속성은 활성화 되었을 때 어떤 동작을 할건지 지정함
속성 값 설명
submit 폼을 서버로 전송합니다.
reset 폼의 입력한 모든 내용을 초기화 시킵니다.
button 버튼 형태만 만들 뿐 자체 기능은 없습니다.

<output>

<output>태그는 계산 결과를 출력하는 태그
- 입력하는 값이 계산 결과라는 것을 브라우저에게 알려줌
- <output>태그를 사용하여 덧셈 가능

<progress>

<progress>태그는 진행 사항을 보여주는 태그
 - <progress>태그의 속성 값은 다음과 같다
속성 설명
min, max 범위의 최솟값과 최댓값을 나타냅니다. 값을 정하지 않으면 0과 1로 간주합니다.
value 범위 내에서 차지하는 값을 나타냅니다.
low *이 정도면 낮다* 라고 할 정도의 값을 지정합니다.
high *이 정도면 높다* 라고 할 정도의 값을 지정합니다.
optimum *이 정도면 적당하다* 라고 할 정도의 값을 지정합니다. optimum 값이 high 값보다 크다면 value 값이 클수록 좋고 optimum 값이 low 값보다 작다면 value 값이 작을수록 좋습니다.

<meter>

<meter>태그는 값이 차지하는 크기를 표시하는 태그
 - <progress>태그와 실행 화면이 유사하여 자주 혼동됨
- <meter>태그의 속성은 다음과 같음
속성 설명
value 작업 진행 상태를 나타내며 부동 소수점으로 표현됩니다. 이 값은 0보타 크거나 같고 max 값보다 작거나 같아야 합니다. 만약 max 값이 지정되지 않았다면 이 값은 1.0보다 작아야 합니다.
max 작업이 완료되려면 얼마나 많은 작업을 해야하는지 부동 소수점으로 표현 됩니다. 이 값은 0보다 커야합니다.
