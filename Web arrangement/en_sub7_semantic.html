The web documents that we often see are different to look at, but if you look inside, they're similar in structure. Oh, my What
are the tags that appear every time you create a web document? In HTML5, you can use the well document's Semantic tags in addition to the previously learned head and body tags when creating the structure You are encouraged to use . Semantic
means "meaningful" and a semantic tag means A defined tag that allows you to easily understand the structure of a page just
by looking at the tag.

HTML5 Semantic Tag

header nav section article aside iframe footer address

<header>

<header>태그는 머리말을 지정하는 태그
 - 주로 페이지 맨 위쪽이나 왼쪽에 삽입
- <form>태그를 사용하여 검색창을 넣거나 <nav>태그를 사용해 사이트 메뉴를 넣음

<nav>

<nav>태그는 문서를 연결하는 내비게이션 링크 영역 태그
 - 다른 사이트의 문서로 이동하는 링크 모음
- footer나 header, aside에도 사용돌 수도 있고 독립적으로도 사용 가능

<section>

<section>태그는 주제별 콘텐츠 영역 태그
 - 문서의 장(chapter, section) 혹은 철을 구성하는 역할
- 일반 문서에 여러 장이 있듯이 웹 페이지에 여러 section 가능
- 제목 태그(<h1>~<h6>)를 사용하여 절 혹은 섹션 주제 기입

<article>

<article>태그는 콘텐츠 내용을 넣는 태그
 - 본문과 연관이 있지만, 독립적인 콘텐츠를 담는 영역
- 혹은 보조 기사, 블로그 포스트, 댓글 등 기타 독립적인 내용
- article에 담는 내용이 많은 경우 여러 section에 둘 수 있음

<aside>

<aside>태그는 본문 이외의 내용 표시
 - 본문에서 약간 벗어난 노트나 팁
- 신문, 잡지에서 주요 기사 옆 관련 기사, 삽입 어구로 표시된 논평 등을 표시
- 페이지의 오른쪽이나 왼쪽에 주로 배치

<iframe>

<iframe>태그는 외부 문서를 삽입하는 태그
 - <iframe>태그에 사용되는 속성은 다음과 같음
속성 설명
width 인라인 프레임의 너비입니다. 픽셀이나 백분율 값으로 표시합니다.
hegiht 인라인 프레임의 높이입니다. 픽셀이나 백분율 값으로 표시합니다.
name 인라인 프레임의 이름입니다.
src 프레임에 표시항 문서의 주소를 지정합니다.
seamless 프레임의 테두리를 없애 마치 본문의 일붗럼 보이도록 만들어 속성 값 없이 seamless라고 쓰면 됩니다. 이 속성은아직 크롬과 사파리에서만 지원합니다.

<footer>

<footer>태그는 제작 정보와 저작권 정보를 표시하는 태그
 - 꼬리말 영역, 주로 저자나 저작권 정보 표시
- <footer>태그 안에 <header>태그나 <section>태그 등 모든 레이아웃 태그를 가용 가능

<address>

<address>태그는 사이트 제작자 정보, 연락처 정보를 나타내는 태그
 - 주로 footer 안에 사용
- 단순한 주소라면 <p>태그를 대신 사용
